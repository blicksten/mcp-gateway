## 1. SAP Picker — landscape parse

- [ ] Open the **SAP Picker** webview (command palette → `MCP Gateway: Open SAP Picker`).
- [ ] All landscape SIDs from `SAPUILandscape.xml` show up (verify count matches `<Service>` entries minus duplicates).
- [ ] Each row carries the right `SID / Client / User` columns and the picker reports a non-zero **Available** count when the KeePass DB is unlocked.
- [ ] Cycle/depth/missing-`<Include>` files surface as warnings in the header, not as a crash (cycle handling in `internal/saplandscape/parser.go::Parse`).

## 2. SAP Picker — `kpMissing` disabled state

- [ ] Pick a landscape SID that is NOT in KeePass (or temporarily rename a KeePass entry so the join misses).
- [ ] The row checkboxes (VSP + GUI) render as `⊘` and are non-interactive.
- [ ] Hover tooltip reads "Add SID to KeePass first — `mcp-ctl credential import` or KeePassXC."
- [ ] Tampering the DOM checkbox state (DevTools → set `disabled=false` and click) does NOT cause an apply call — host-side `buildOpsList` skips the row.

## 3. SAP Picker — batch Apply success (mixed add + remove)

- [ ] Tick VSP for one new SID and untick GUI for one already-registered SID.
- [ ] Press **Apply**. Status bar shows `MCP: N/M`.
- [ ] Each row transitions through `pending` → `in_progress` → terminal status (`config_added_running` / `removed`).
- [ ] Daemon log shows exactly **one** `TriggerPluginRegen` + one `RebuildTools` invocation for the entire batch (single-regen invariant — R-26 / X2). Verify via `mcp-ctl servers list` returning the union of pre-batch + new entries on a single refresh tick.

## 4. SAP Picker — retry-failed-rows + force-kill on orphan

- [ ] Take down the daemon's child process for one row mid-apply (`kill -9 <pid>` on POSIX, Task Manager → End Task on Windows).
- [ ] Row transitions to `removed_with_orphan`. **Force kill** button surfaces.
- [ ] Click **Force kill** → confirm dialog → row clears to `removed`.
- [ ] Click **Retry failed rows** on a different failed row → only that row re-runs; succeeded rows stay green.

## 5. Import-from-Claude — all 3 sources

- [ ] Open the **Import from Claude** webview.
- [ ] Switch source `cc_global` → `cc_project` (with a workspace open) → `desktop`.
- [ ] Each switch refreshes the row list from the corresponding file (`~/.claude.json`, `<workspace>/.mcp.json`, `%APPDATA%\Claude\claude_desktop_config.json` on Windows / `$HOME/Library/Application Support/Claude/claude_desktop_config.json` on macOS / `$XDG_CONFIG_HOME/Claude/claude_desktop_config.json` on Linux).
- [ ] Empty / non-existent source files show an empty list with a hint banner — no crash.

## 6. Import-from-Claude — copy + skip happy path

- [ ] Tick two rows in `cc_global`. Action `copy`. Conflict `skip`.
- [ ] Press **Preview** → modal lists both rows with action `copy`, status `(planned add)` and the resolved command path.
- [ ] Press **Apply** → both rows show `applied`.
- [ ] `mcp-ctl servers list` shows both new entries.

## 7. Import-from-Claude — move + overwrite UX warning (R-23)

- [ ] Tick one row, action `move`, conflict `overwrite`.
- [ ] Toolbar surfaces the red banner: **⚠ Move + Overwrite mutates source AND discards local edits**.
- [ ] Press **Preview** → modal shows action `move` highlighted; warning repeated in modal.
- [ ] Press **Apply** → row shows `applied`; the source file no longer contains that row; gateway has it; provenance sidecar `~/.mcp-gateway/claude-imported.json` has a new record with `action: "move"` for that name.

## 8. Restart-required toast (Settings webview, R-29 / X5)

- [ ] Open **Settings** webview, change `mcpGateway.daemonPath` (or any other restart-required key — `apiUrl`, `authTokenPath`, `claudeConfigSync.enabled`, `claudeConfigSync.namespacePrefix`, `claudeConfigSync.path`, `claudeConfigSync.aggregateEntryName`).
- [ ] Press **Save**. Toast surfaces: "Settings saved. Restart daemon to apply changes to: <list>." with `[Restart Daemon]` and `[Later]` buttons.
- [ ] Click `[Restart Daemon]` → daemon stops + spawns fresh (poll `mcp-ctl daemon status`).
- [ ] Save with NO restart-required keys changed → no toast.

## 9. Settings sticky scroll @ 800 px viewport

- [ ] Resize the VSCode window so the Settings webview viewport is ≤800 px tall.
- [ ] Header (filter + import row) sticks at top; footer (Save / Cancel) sticks at bottom.
- [ ] All 25+ fields visible via vertical scroll; no horizontal scroll.

## 10. Browse fallback chain (T-C.2)

- [ ] In Settings, find a path field with an empty value (e.g. `keepassPath`).
- [ ] Click **Browse** → file dialog opens at `os.homedir()` (currentValue empty + parentDir empty → fall back to home).
- [ ] Set the field to a non-existent path like `C:\does\not\exist\sub\file.kdbx` (or `/tmp/missing/file.kdbx` on POSIX). Click **Browse** again → dialog opens at the parent dir if it exists, else at home.
- [ ] Set the field to a valid existing file path. Click **Browse** → dialog opens at that file's parent dir.

## 11. Version-skew gate (R-24)

- [ ] Run `mcp-ctl daemon stop` and start an old daemon (≤v1.5) on the same port (use the previous binary if available; otherwise mock by patching `/api/v1/claude-code/compat-matrix` to return a missing entry).
- [ ] Open the **Import from Claude** webview → status banner shows "Upgrade gateway to v1.6+ to import" instead of attempting the call.
- [ ] Restore the v1.9.0 daemon → banner clears; webview functions normally.

## 12. Provenance badge re-import

- [ ] Import a row from `cc_global` (action `copy`).
- [ ] Open the picker again, same source. The same row now displays the `◊ previously imported` badge with the original timestamp on hover.
- [ ] Re-import the same row a second time (overwrite). The badge stays + tooltip surfaces both timestamps (latest first per `PreviouslyImported` sort).

## 13. Q3.4 — multi-monitor `showOpenDialog` quirk (Windows)

- [ ] On a Windows multi-monitor setup, with VSCode on monitor 2, click **Browse** in Settings.
- [ ] The dialog may open on the primary monitor (monitor 1) instead of next to VSCode. **This is upstream VSCode behaviour** — drag the dialog to the correct monitor or move VSCode to the primary monitor before clicking Browse. Documented in README §"Known limitations".
- [ ] No crash, no broken focus — just a UX quirk that we surface in the README rather than work around.
